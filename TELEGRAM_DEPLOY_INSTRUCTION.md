## Что такое Telegram Mini App

**Telegram Mini App** — это полноценное веб-приложение, которое открывается внутри Telegram. Это как приложение внутри приложения!

**Пример:** Когда вы открываете бота и нажимаете кнопку "BUY NOW", открывается целое веб-приложение с товарами, корзиной, оформлением заказа — это и есть Mini App.

---

## Что нужно сделать (общая схема)

Вам нужно сделать следующее:

1. ✅ Создать Telegram бота
2. ✅ Настроить **сервер** (бэкенд) — он будет получать сообщения от пользователей
3. ✅ Настроить **клиент** (фронтенд) — это само приложение с товарами
4. ✅ Настроить **админ-панель** — панель управления для вас (просмотр заказов, товаров)
5. ✅ Загрузить всё это на интернет-хостинг (Vercel)
6. ✅ Подключить Mini App к боту

**Схема работы:**
```
Пользователь → Бот (сервер) → Mini App (клиент) → Админ-панель
```

## Шаг 1: Создание Telegram бота

### 1.1 Откройте Telegram и найдите бота @BotFather

- Откройте Telegram
- В поиске введите `@BotFather`
- Нажмите "Начать" или "/start"

### 1.2 Создайте нового бота

В чат с @BotFather отправьте команду:
```
/newbot
```

BotFather спросит:
1. **Название бота** — например: "Мой Магазин" (любое название)
2. **Имя пользователя** — должно заканчиваться на `_bot`, например: `myshop_bot`

Пример:
```
Bot name: Мой Магазин
Username: myshop_bot
```

### 1.3 Получите токен бота

BotFather пришлёт вам **токен** — это длинная строка вида:
```
123456789:ABCdefGHIjklMNOpqrsTUVwxyz1234567890
```

**СОХРАНИТЕ ЭТОТ ТОКЕН!** Вы будете использовать его дальше. Скопируйте его в блокнот.

## Шаг 2: Настройка серверной части (Backend)

### 2.1 Установите Node.js

**Важно!** Ваш компьютер должен иметь установленный Node.js.

**Проверьте, установлен ли Node.js:**
1. Откройте Терминал (на Mac: Launchpad → Терминал)
2. Введите команду:
```bash
node --version
```

Если видите версию (например, `v20.10.0`), всё хорошо. Если нет, установите Node.js:
- Скачайте с [nodejs.org](https://nodejs.org)
- Установите LTS-версию (рекомендуемую)

### 2.2 Перейдите в папку проекта

В Терминале выполните:
```bash
cd /Users/viktorlavrusko/IdeaProjects/tgcart
```

### 2.3 Перейдите в папку сервера
```bash
cd server
```

### 2.4 Создайте файл настроек .env

**Очень важно:** В папке `server/` должен быть файл `.env` (без расширения).

**Как создать:**
1. Откройте текстовый редактор (TextEdit, VS Code, Sublime Text)
2. Вставьте этот текст (замените токен на свой):

```env
BOT_TOKEN=1341437044:AAEnJflnTJCaUWCkLv0oxip4pFNISMVpneX
ADMIN_CHAT_ID=123456789
CLIENT_APP_URL=https://your-client-url.com
ADMIN_PANEL_URL=https://your-admin-panel-url.com
PORT=5000
```

**Что нужно изменить:**

1. **BOT_TOKEN** — замените на ваш токен от BotFather (из Шага 1.3)
2. **ADMIN_CHAT_ID** — ваш ID в Telegram
   - Чтобы получить ID: откройте бота @userinfobot в Telegram
   - Нажмите "Старт" или "/start"
   - Бот пришлёт ваш ID — скопируйте его
3. **CLIENT_APP_URL** — пока оставьте заглушку, мы заполним после деплоя
4. **ADMIN_PANEL_URL** — пока оставьте заглушку, мы заполним после деплоя
5. **PORT** — оставьте как есть (5000)

**Сохраните файл в папке `server/` под названием `.env`**

**ВАЖНО:** Файл должен называться именно `.env` (с точкой в начале). На Mac файлы с точкой могут скрываться — проверьте, что он есть:
- В Finder нажмите `Cmd + Shift + .` (точка) — это покажет скрытые файлы
- Или используйте терминал:
```bash
ls -la
```
Должна появиться строка с `.env`

### 2.5 Установите зависимости сервера

В Терминале (в папке `server/`):
```bash
npm install
```

Это установит все необходимые библиотеки. Подождите, пока завершится.

### 2.6 Проверьте работу локально (опционально)

В Терминале (в папке `server/`):
```bash
npm start
```

Сервер запустится на порту 5000. Вы увидите сообщение:
```
Server listening on port 5000
```

Для тестирования можно открыть в браузере: `http://localhost:5000/health`

Вы должны увидеть: `{"status":"ok"}`

**Остановите сервер:** Нажмите `Ctrl + C` в терминале.

## Шаг 5: Деплой на хостинг (Vercel)

Теперь мы загрузим всё это в интернет, чтобы приложение работало постоянно.

### 5.1 Зарегистрируйтесь на Vercel

1. Откройте [vercel.com](https://vercel.com)
2. Нажмите "Sign Up" или "Continue with GitHub"
3. Войдите с помощью GitHub (рекомендуется)

### 5.2 Деплой Backend (сервера)

#### 5.2.1 Загрузите сервер на GitHub (если ещё не загружен)

**Важно:** Vercel берёт код из GitHub. Убедитесь, что код есть на GitHub.

**Как проверить:**
1. Откройте [github.com](https://github.com)
2. Войдите в свой аккаунт
3. Проверьте, есть ли ваш репозиторий tgcart

**Если код ещё не на GitHub:**

В Терминале (в папке tgcart):
```bash
# Инициализируйте git (если ещё не инициализирован)
git init

# Добавьте все файлы
git add .

# Сделайте коммит
git commit -m "Initial commit"

# Подключите репозиторий (замените на ваш GitHub URL)
git remote add origin https://github.com/ВАШ_ИМЯ/tgcart.git

# Загрузите код
git push -u origin main
```

#### 5.2.2 Деплой через Vercel Dashboard

1. Откройте [vercel.com](https://vercel.com)
2. Нажмите "Add New Project"
3. Выберите "Import Git Repository"
4. Найдите и выберите репозиторий `tgcart`
5. Нажмите "Import"

#### 5.2.3 Настройка переменных окружения для Backend

После импорта Vercel спросит переменные окружения:

**Для Backend:**
- **Root Directory:** `server`
- Переменные:
  - `BOT_TOKEN` — ваш токен от BotFather
  - `ADMIN_CHAT_ID` — ваш ID из @userinfobot
  - `CLIENT_APP_URL` — оставьте пустым или укажите будущий URL клиента (заполним позже)
  - `ADMIN_PANEL_URL` — оставьте пустым или укажите будущий URL админки (заполним позже)

Нажмите "Deploy"

**Дождитесь завершения деплоя.** Vercel покажет URL вида:
```
https://tgcart-server.vercel.app
```

**СКОПИРУЙТЕ ЭТОТ URL!** Он понадобится дальше.

#### 5.2.4 Установите переменные окружения через интерфейс

Если вы не указали переменные при деплое:

1. Зайдите в проект на Vercel
2. Нажмите "Settings" (шестерёнка)
3. Вкладка "Environment Variables"
4. Добавьте переменные:
   - `BOT_TOKEN`
   - `ADMIN_CHAT_ID`
   - `CLIENT_APP_URL` (заполним после деплоя клиента)
   - `ADMIN_PANEL_URL` (заполним после деплоя админки)

5. Нажмите "Save"

### 5.3 Деплой Frontend (клиента)

#### 5.3.1 Создайте новый проект для клиента

1. На Vercel нажмите "Add New Project"
2. Выберите "Import Git Repository"
3. Найдите репозиторий `tgcart`
4. **Когда спросит Root Directory, укажите:** `client`
5. Переменные окружения:
   - `VITE_APP_BACKEND_URL` — URL вашего backend (https://tgcart-server.vercel.app)

6. Нажмите "Deploy"

**Дождитесь завершения.** Vercel покажет URL вида:
```
https://tgcart.vercel.app
```

**СКОПИРУЙТЕ ЭТОТ URL!** Он понадобится для backend.

### 5.4 Деплой Admin Panel (админ-панели)

#### 5.4.1 Создайте новый проект для админ-панели

1. На Vercel нажмите "Add New Project"
2. Выберите "Import Git Repository"
3. Найдите репозиторий `tgcart`
4. **Когда спросит Root Directory, укажите:** `admin-panel`
5. Переменные окружения:
   - `VITE_APP_BACKEND_URL` — URL вашего backend (https://tgcart-server.vercel.app)

6. Нажмите "Deploy"

**Дождитесь завершения.** Vercel покажет URL вида:
```
https://tgcart-admin.vercel.app
```

**СКОПИРУЙТЕ ЭТОТ URL!** Он понадобится для backend.

## Шаг 6: Обновление URL-ов в Backend

После деплоя клиента и админ-панели у вас есть 3 URL:

1. **Backend URL:** `https://tgcart-server.vercel.app`
2. **Client URL:** `https://tgcart.vercel.app`
3. **Admin Panel URL:** `https://tgcart-admin.vercel.app`

Теперь нужно обновить переменные окружения в Backend:

### 6.1 Обновите CLIENT_APP_URL в Backend

1. Зайдите в проект Backend на Vercel
2. Нажмите "Settings"
3. Вкладка "Environment Variables"
4. Найдите переменную `CLIENT_APP_URL`
5. Измените её на URL клиента (например, `https://tgcart.vercel.app`)
6. Нажмите "Save"

### 6.2 Обновите ADMIN_PANEL_URL в Backend

1. В той же вкладке "Environment Variables"
2. Найдите переменную `ADMIN_PANEL_URL`
3. Измените её на URL админ-панели (например, `https://tgcart-admin.vercel.app`)
4. Нажмите "Save"

### 6.3 Передеплой Backend

После изменения переменных окружения Vercel автоматически пересоберёт проект. Подождите несколько секунд.

## Часто встречающиеся проблемы и решения

### Проблема 1: "Oops! Telegram initData is missing!"

**Причина:** Приложение открывается не внутри Telegram.

**Решение:**
- Убедитесь, что открываете Mini App через кнопку меню в боте
- Не открывайте прямую ссылку в браузере (это не будет работать)

### Проблема 2: Кнопка "BUY NOW" не открывает приложение

**Причина:** Menu Button не настроен или настроен неправильно.

**Решение:**
1. Проверьте, что Menu Button настроен через BotFather
2. Убедитесь, что URL правильный (начинается с https)
3. Убедитесь, что URL деплоен и доступен

### Проблема 3: "Access denied" в админ-панели

**Причина:** Your ID не совпадает с `ADMIN_CHAT_ID`.

**Решение:**
1. Узнайте свой ID через @userinfobot
2. Обновите `ADMIN_CHAT_ID` в переменных окружения Backend
3. Передеплойте Backend

### Проблема 4: Сервер не запускается

**Причина:** Отсутствует файл `.env` или он неправильно настроен.

**Решение:**
1. Убедитесь, что файл `.env` есть в папке `server/`
2. Проверьте, что переменные заполнены
3. Убедитесь, что нет ошибок в форматировании

### Проблема 5: Клиент не может подключиться к серверу

**Причина:** `VITE_APP_BACKEND_URL` неправильный.

**Решение:**
1. Проверьте, что `VITE_APP_BACKEND_URL` указывает на правильный URL backend
2. Убедитесь, что URL деплоен и доступен
3. Проверьте, что backend не выдаёт ошибок (посмотрите логи на Vercel)

### Проблема 6: Картинки товаров не грузятся

**Причина:** Проблемы с CDN DummyJSON (временная).

**Решение:**
- Подождите несколько минут
- Обновите страницу
- В продакшене можно заменить DummyJSON на свой API
