# 📋 Инструкция по настройке уведомлений администратору

## 🔧 Настройка ADMIN_CHAT_ID

### Шаг 1: Получите ID вашего чата

1. Откройте бота **@userinfobot** в Telegram
2. Нажмите `/start`
3. Бот пришлет вам ваш ID чата

**Пример:**
```
Your chat ID: 123456789
```

### Шаг 2: Добавьте ID в .env файл backend

Откройте файл `server/.env` и добавьте строку:

```env
ADMIN_CHAT_ID=123456789
```

> **Важно:** Это должен быть именно ваш личный чат (начинается с 1, 2 или 3), а не групповой чат.

### Шаг 3: Перезапустите backend

```bash
cd server
npm run start
```

---

## 📨 Как это работает

Когда пользователь оформляет заказ:

1. В приложении нажимается кнопка "ОФОРМИТЬ ЗАКАЗ"
2. Backend получает запрос на `/order-notification`
3. Сообщение отправляется в ваш чат с ID из `ADMIN_CHAT_ID`

### Пример сообщения администратору:

```
🛒 Новый заказ!

👤 Пользователь: @username
🆔 ID: 123456789

📦 Товар: iPhone 15
💰 Цена: $999 (15% скидка)
🔢 Количество: 2
📊 Итого: $1698.30

⏳ Статус: Новый заказ (нужно связаться с покупателем)
```

## ⚠️ Возможные проблемы

### "ADMIN_CHAT_ID не настроен"
- Проверьте, что переменная установлена в `.env` файле
- Перезапустите backend после добавления переменной

### "Уведомление не приходит"
- Проверьте ID чата (должен начинаться с 1, 2 или 3)
- Убедитесь, что вы начали диалог с ботом
- Проверьте логи сервера на ошибки

### "Forbidden: bot was blocked by the user"
- Вы заблокировали бота, разблокируйте его
- Или используйте другого администратора

---

## 💡 Советы

1. **Создайте отдельного бота для уведомлений** - лучше использовать отдельного бота для админки
2. **Используйте групповой чат** - можно добавить в группу и отправлять туда (нужно получить ID группы)
3. **Логируйте заказы** - добавьте логирование в базу данных одновременно с отправкой уведомления
