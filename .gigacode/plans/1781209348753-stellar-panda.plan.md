# План добавления админ-панели в TGCart

## Текущее состояние
- Проект представляет собой Telegram Mini App для e-commerce
- Администратор получает только уведомления о заказах через Telegram
- Нет возможности управлять заказами и продуктовой базой из веб-интерфейса

## Требования
Добавить полноценную админ-панель с двумя основными разделами:
1. **Управление заказами** - просмотр всех заказов, обновление статуса
2. **Управление продуктами** - CRUD операции (создание, чтение, обновление, удаление)

## Архитектура решения

### 1. Серверная часть (`server/`)

#### Новые эндпоинты:

**Управление заказами:**
- `GET /api/orders` - получить все заказы из файла
- `PUT /api/orders/:id/status` - обновить статус заказа

**Управление продуктами:**
- `GET /api/products` - получить все продукты из JSON файла
- `POST /api/products` - добавить новый продукт
- `PUT /api/products/:id` - обновить продукт
- `DELETE /api/products/:id` - удалить продукт

**Аутентификация:**
- `POST /admin/login` - проверка прав администратора через Telegram
- Защита всех эндпоинтов админки через middleware

**Хранение данных:**
- Заказы сохранять в `server/data/orders.json` (json file)
- Продукты считывать/записывать в `server/data/products.json`
- Создать структуру папок для хранения данных

### 2. Клиентская часть (`admin-panel/`)

#### Структура:
- Создать отдельное React приложение в папке `admin-panel/`
- Использовать Vite + React + Tailwind CSS
- Интеграция с Telegram WebApp API
- Аутентификация через Telegram

#### Компоненты:

**AdminLayout.jsx** - основной layout с навигацией
**Login.jsx** - экран авторизации администратора
**OrdersList.jsx** - список всех заказов
**OrdersDetail.jsx** - детальная информация о заказе
**ProductsList.jsx** - список всех продуктов
**ProductForm.jsx** - форма добавления/редактирования продукта

### 3. Данные заказов

**Структура заказа в `orders.json`:**
```json
{
  "id": "order_12345",
  "userId": 123456789,
  "username": "@username",
  "product": { ... },
  "count": 2,
  "total": "199.98",
  "status": "new", // new, confirmed, shipped, completed, cancelled
  "createdAt": "2026-06-11T10:30:00Z"
}
```

## Реализация

### Шаг 1: Настройка сервера
1. Создать папку `server/data/` для хранения JSON файлов
2. Создать `server/data/orders.json` с пустым массивом
3. Создать `server/data/products.json` с копией текущих продуктов
4. Добавить новые эндпоинты в `server/index.js`
5. Создать middleware для аутентификации администратора
6. Добавить функции для работы с JSON файлами

### Шаг 2: Создание admin-panel
1. Создать папку `admin-panel/`
2. Инициализировать Vite React приложение
3. Настроить Tailwind CSS
4. Создать компоненты для админ-панели
5. Настроить API запросы к серверу

### Шаг 3: Интеграция
1. Добавить кнопку "Admin Panel" в главное меню бота
2. Настроить URL для админ-панели в `.env` файле
3. Протестировать все функции

### Шаг 4: Безопасность
1. Аутентификация только через Telegram (проверка данных из Mini App)
2. Проверка прав администратора по ADMIN_CHAT_ID
3. Валидация входных данных
4. Обработка ошибок

## Файлы для создания/модификации

**Создать:**
- `admin-panel/` - директория с React приложением
- `server/data/orders.json` - хранение заказов
- `server/data/products.json` - копия продуктов для редактирования
- `admin-panel/src/components/AdminLayout.jsx`
- `admin-panel/src/components/Login.jsx`
- `admin-panel/src/components/OrdersList.jsx`
- `admin-panel/src/components/OrdersDetail.jsx`
- `admin-panel/src/components/ProductsList.jsx`
- `admin-panel/src/components/ProductForm.jsx`

**Модифицировать:**
- `server/index.js` - добавить новые эндпоинты и middleware
- `server/.env` - добавить URL для админ-панели
- `server/const/messages.js` - добавить сообщение с кнопкой админки
- `server/utils/bot-methods.js` - добавить функцию проверки прав админа

## URL конфигурация

В `.env` файле server:
```
ADMIN_PANEL_URL=https://your-admin-panel-url.com
```

## Тестирование

1. Запустить сервер и проверить эндпоинты через curl/Postman
2. Запустить admin-panel и проверить авторизацию
3. Создать тестовый заказ и проверить его появление в админке
4. Обновить статус заказа через админку
5. Добавить/редактировать/удалить продукт
6. Проверить сохранение изменений
